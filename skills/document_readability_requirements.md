# 文档可读性要求

## 概述
为确保所有文档（尤其是 .md 和 .json 文件）具有良好的可读性，便于人类和 AI 助手快速理解和使用，制定以下规范。

## JSON 文件可读性要求

### 格式规范
- 使用 2 空格缩进
- 每个键值对单独一行
- 数组元素分行显示
- 字符串必须使用双引号
- 不允许出现压缩的单行 JSON

### 内容组织
- 使用清晰的层次结构
- 相关字段逻辑分组
- 为复杂配置添加注释说明
- 数值和单位保持一致格式

### 示例
```json
{
  "project_name": "示例项目",
  "version": "1.0.0",
  "settings": {
    "timer": {
      "mode": "16-bit",
      "frequency": "750000Hz"
    }
  }
}
```

## Markdown 文件可读性要求

### 结构规范
- 使用层级标题（#、##、###）清晰划分内容
- 段落之间保留空行
- 重要概念使用粗体或强调标记
- 代码块指定语言类型

### 内容组织
- 采用"总-分-总"结构
- 使用列表和表格组织信息
- 为技术术语提供简要说明
- 关键数据和配置突出显示

### 示例
```markdown
# 标题

## 子标题

- 要点一
- 要点二

**重要概念**：简要说明

```c
代码示例
```
```

## 通用要求

1. **语言一致性**：文档语言统一（中文或英文）
2. **命名规范**：文件名使用下划线或短横线分隔
3. **更新记录**：重要文档应包含最后更新日期
4. **注释清晰**：技术文档必须包含必要的注释和说明

## 特殊说明

对于项目相关的技术文档：
- 必须包含背景信息、目标和当前状态
- 配置文件必须与实际代码保持同步
- 所有修改必须注明原因和日期
