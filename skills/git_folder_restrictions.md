# Git 文件夹操作限制

## 重要警告

你拥有修改项目源文件的权限，但严禁读取、修改或删除项目根目录下的 `.git` 文件夹及其任何内容。这是项目的版本控制核心，任何对该文件夹的操作都会导致严重的系统错误。

## 禁止操作

- ❌ **读取** `.git` 文件夹及其子目录中的任何文件
- ❌ **修改** `.git` 文件夹中的任何配置文件或数据
- ❌ **删除** `.git` 文件夹或其中的任何内容
- ❌ **移动** 或 **重命名** `.git` 文件夹

## 允许操作

- ✅ 使用 `git` 命令行工具进行版本控制操作
- ✅ 使用标准 Git 工作流（add, commit, push, pull 等）
- ✅ 修改项目源文件（`.git` 文件夹之外的内容）

## 后果说明

任何违反此限制的操作都可能导致：
- Git 版本历史丢失
- 项目状态无法恢复
- 团队协作中断
- 需要重新初始化整个仓库

## 安全措施

所有文件操作工具（`read_file`、`write_file`、`replace` 等）在操作 `.git` 文件夹时都会被严格限制。
